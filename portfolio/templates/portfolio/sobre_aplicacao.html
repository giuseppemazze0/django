Sobre esta Aplicação

1. Arquitetura MVT do Django

Esta aplicação foi desenvolvida recorrendo à arquitetura Model-View-Template (MVT), utilizada pelo Django.

Componentes

  • Model – representa os dados da aplicação e comunica com a base de dados.
  • View – recebe os pedidos do utilizador, processa a lógica da aplicação e envia informação para o Template.
  • Template – apresenta a informação ao utilizador através de HTML.

Fluxo de Informação

Utilizador → URL → View → Model → Base de Dados → View → Template → Navegador
